MODULE 2

AI 視覺資產:
從許願到可控

學完:生出風格一致、可重現的活動視覺——主視覺、貼文圖、OG 分享圖。

許願 vs 下規格

「幫我畫咖啡店開幕的圖」每次都在重新抽籤。
行銷要的是一套視覺,不是一張圖。

視覺規格卡:六個欄位

欄位範例
主體一杯拿鐵在木質吧台上,蒸氣上升
風格日系極簡攝影,底片感
光線清晨柔和側光,窗光
構圖主體置右,左側留白放標題
配色深墨綠 + 奶油白 + 木質棕
負面不要文字、不要人臉、不要過度銳利

產第二張圖只換「主體」「構圖」,其他欄位不動,整套視覺自然成系列。

實證:本課 demo 的 hero 就是這樣生的

主體:山嵐間的咖啡店窗景,熱拿鐵在窗台,蒸氣清楚上升,旁邊兩三顆咖啡豆
風格:日系極簡商業攝影,底片顆粒感,安靜高級
光線:清晨窗光,柔和,從左側來
構圖:16:9 橫式,杯子置於右三分之一,左側大量留白(要疊活動標題)
配色:深墨綠、奶油白、木質棕,低飽和
避免:任何文字、人臉、logo、過度 HDR 銳利感

demos/01-landing/assets/hero.jpg — 一次生成到位,因為規格卡把決定權留給自己。

一次生一個用途

  • 主視覺 16:9,構圖預留標題位
  • IG 貼文 1:1 方形
  • OG 分享圖 1200x630,主體靠中(各平台會裁切)
  • 微調用追加:「同一張,光線改黃昏」,不要重寫整個 prompt

文字疊圖的正確做法

  • 生圖模型寫中文幾乎必錯
  • 圖只負責氛圍(所以構圖要留白)
  • 文字用網頁疊上去:「hero 換這張圖,標題疊左側留白,白字加細陰影」
  • 文案隨時能改,不用重新生圖

常見坑

  • 每張圖重新發明風格 → 先卡死規格卡的風格欄位
  • OG 圖留白不足 → 重要元素放中央 80% 安全區
  • 直接叫模型畫字 → 字交給網頁層
  • 圖太大 → 壓到 500KB 以下,行動網路 3 秒打不開人就走了
