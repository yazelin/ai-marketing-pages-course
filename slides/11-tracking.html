MODULE 11

成效追蹤:
裝分析與廣告 Pixel

知道多少人來、有沒有報名、廣告有沒有效。都是貼一段 script 的事。

為什麼一定要裝

沒有分析,你只是憑感覺;
裝了,下次活動才有依據優化。

一、網站分析(每頁都該裝)

<!-- Google Analytics 4:貼進 <head> -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXX');
</script>

換成你自己的 G-XXXXXXX。在意隱私可改 Plausible / Umami,同樣貼一段。

二、追「報名成功」(關鍵)

  • 只知道「有人來」不夠,要知道「幾個人真的報名」
  • 報名表單送出成功時送事件:gtag('event', 'signup', …)
  • 後台才算得出轉換率(報名數 ÷ 來訪數)

三、廣告 Pixel(要投廣告才裝)

  • 投 FB/IG 廣告必裝 Meta Pixel,否則追不到哪個廣告帶來報名
  • 裝了才能再行銷(把廣告投給來過卻沒報名的人)
  • 報名成功加 fbq('track', 'Lead') 追轉換;不投廣告就不用裝

常見坑

  • 裝了沒驗證 → 開頁面在 GA「即時」看到自己這次造訪才算通
  • 只追流量、不追轉換 → 一定要設「報名成功」事件
  • Pixel 只 init 不 track → 後台看不到轉換、做不了再行銷
  • 追蹤碼貼錯位置 → 主代碼放 head;track 放在動作完成那一刻
  • 忘了隱私揭露 → 蒐集資料卻沒說,法規與信任有風險
做完把頁貼進健檢器,「追蹤」那格應變綠
