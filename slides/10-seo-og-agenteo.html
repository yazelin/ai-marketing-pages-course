MODULE 10

被看見:
SEO、社群卡、給 AI 看的頁面

做得再好的活動頁,沒人看見等於沒做。

頁面被看見的三條路

  • 有人 Google 搜尋 → SEO
  • 有人把連結貼到 LINE / FB / IG → OG 社群分享卡
  • 有人問 ChatGPT / Claude / Perplexity → Agent EO(給 AI 看的頁面)

第三條是 2026 越來越重要的新管道:很多人不 Google 了,直接問 AI。

一、SEO 基本功

  • title 每頁不同、放關鍵字 + 品牌
  • meta description 一兩句,搜尋結果摘要,寫得讓人想點
  • sitemap.xml 列所有公開頁,丟 Search Console
  • 不想被收錄的頁(後台/感謝頁)用 noindex 擋

二、社群分享卡 OG

<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:image" content="https://你的網域/og.jpg">  <!-- 完整網址! -->
<meta name="twitter:card" content="summary_large_image">
  • og:image 必須完整 https 網址,相對路徑 LINE/FB 抓不到
  • 圖 1200x630、壓 200KB 內;別留 example.com 假網址(本課第一版就犯過,分享破圖)
  • 改了要用 FB Sharing Debugger 強制重抓(平台會快取)

Agent EO

越來越多人問 AI 而不是 Google。
要讓 AI 講得出你的活動。

三、Agent EO:給 AI 看的頁面

  • JSON-LD(schema.org) — 機器讀得懂的格式描述「這頁是什麼」;活動用 Event、課程用 Course、店家用 LocalBusiness。AI 與 Google 都吃
  • llms.txt — 放網站根的純文字說明書,專門寫給 LLM:你是誰 + 重要連結
  • 重要資訊用文字寫 — 日期/地點/優惠別只放圖裡,圖裡的字 AI 讀不到

驗收

  • SEO:Google 搜「site:你的網址」;Search Console 提交 sitemap
  • OG:貼到 LINE 看預覽卡;FB Sharing Debugger 看抓到什麼
  • Agent EO:把網址貼給 ChatGPT/Claude 問「這頁在講什麼活動」,看講得對不對;Rich Results Test 驗 JSON-LD

常見坑

  • og:image 相對路徑/假網址 → 分享破圖,必完整 https
  • 改了 OG 沒重抓 → 平台快取,用 debugger
  • JSON-LD 跟頁面對不上 → 會被懲罰,要一致
  • 重要資訊只在圖裡 → AI 讀不到,用文字也寫一遍
  • 專案頁 robots/llms 在子路徑 → 完整 SEO 要自訂網域(模組 5)
對照:本課整站就是案例(每頁 OG+JSON-LD、sitemap、llms.txt、後台 noindex)
